What the app actually does

A single-file, browser-only label maker. Pick a workflow, fill a form, and it mints a human-readable lot ID, renders a live SVG label sized to the label stock, and prints it or exports a 300-DPI PNG — backed by a genetics catalog, sequence counters, printed-lot records, and a lineage graph. Label engine

hand-written SVG strings · no library

Labels are SVG documents built as template literals — no PDF/QR/canvas/SVG library. Each template owns render, renderSimple, and fit (auto-shrink via SVG textLength; needs a live DOM).

Persistence & auth

db.js · local-first + cloud

localStorage source of truth

Per-user namespaced keys ql_u:<user>:<slot>. Slots: cfg, counters, form, genetics, lot_records, lineage. Legacy flat keys imported once, then backup.

Supabase KV sync built

Postgres table ql_store (user_id / slot / data / updated_at), debounced push + pull-on-login. Synced: cfg, counters, genetics, lots, lineage — form is device-local.

Supabase Auth built

Synthetic email <user>@quicklabel.app; per-user login, token refresh, force / self-serve (claim_password) reset; RLS on auth.uid.

Known bugs & footguns

documented, not fixed

⚠︎ Carry these into the port

  1. Inventory prefix mismatch. The inventory filters use badges GS (grain) and AP (agar), but those workflows mint GL- and AL- — so grain & agar lots never match their own filter chips.
  2. Print page size is hardcoded. @page is fixed at 2.25×1.25 in regardless of the selected printer, so the Merryhome/D11 prints at the wrong page size.
  3. COMPLEX_INGEST_TYPES is declared but effectively unused; pre-prefix lot counters are orphaned (no migration).
  4. cfg.codes ⇄ genetics is kept in sync bidirectionally on every login — two sources of one catalog to reconcile when porting.

The domain is a mushroom grower model — genetics, lot codes, lineage, counters, free-text potency map straight across. Keep the SVG render/template/fit layer, swap the data source to your entities, add QR into the reserved box, and replace the print/export layer.
